# Releasing Niacin

One sandboxed build, two distribution channels:

| Channel              | Signing identity        | Artifact                                  |
| -------------------- | ----------------------- | ----------------------------------------- |
| GitHub Releases      | Developer ID Application + Installer | `.dmg` + `.pkg`, notarized + stapled |
| Mac App Store        | Apple Distribution      | Archive uploaded via Xcode Organizer      |

Same `niacin` target, same `Release` configuration, same `com.oldsalt.niacin` bundle ID. Only the signing identity and distribution mechanism differ. Both builds have App Sandbox on and the same entitlement set.

---

## One-time setup

Re-run after macOS reinstalls or DerivedData wipes:

1. **Developer ID certs in Keychain** (for the GitHub channel). `security find-identity -v | grep "Developer ID"` should show both `Developer ID Application: …` and `Developer ID Installer: …`. Add via Xcode → Settings → Accounts → Manage Certificates.
2. **Apple Distribution cert in Keychain** (for the App Store channel). Same Xcode Manage Certificates dialog.
3. **Notarization profile** (for the GitHub channel). `xcrun notarytool store-credentials niacin-notary --apple-id YOUR_APPLE_ID --team-id 346JJCHZP7` — prompts for an app-specific password (generate at appleid.apple.com → Sign-In & Security → App-Specific Passwords).
4. **App Store Connect record** for bundle ID `com.oldsalt.niacin`. Set the subtitle without Apple trademarks: e.g. **"Keep your computer awake."** (≤30 chars, no "Mac" / "macOS" — those trigger 5.2.5 IP rejections).

---

## GitHub release workflow

1. **Bump version.** Edit `MARKETING_VERSION` and `CURRENT_PROJECT_VERSION` in `niacin.xcodeproj/project.pbxproj` (each appears six times — search-and-replace both numbers). Commit:
   ```
   git commit -am "Bump version to X.Y (build N)"
   ```

2. **Build, sign, notarize, package.** From the repo root:
   ```
   ./scripts/release.sh X.Y
   ```
   Takes 3–15 minutes — two notarization rounds (`.dmg`, `.pkg`). The script verifies the version arg matches `MARKETING_VERSION`, so it'll bail loudly if you forgot step 1.

3. **Tag and publish to GitHub.** The script prints the exact commands; copy-paste:
   ```
   git tag -a vX.Y -m "Niacin X.Y"
   git push origin vX.Y
   gh release create vX.Y \
       --title "Niacin X.Y" \
       --generate-notes \
       build/release/artifacts/niacin-X.Y.dmg \
       build/release/artifacts/niacin-X.Y.pkg \
       build/release/artifacts/SHA256SUMS
   ```

## Mac App Store workflow

Same version bump (step 1 above). Then:

1. **Archive in Xcode.** Product → Archive (default scheme + `Release` configuration).
2. **Distribute App** from the Organizer window → App Store Connect → Upload. Xcode validates the signature, sandbox entitlements, and uploads to App Store Connect.
3. In App Store Connect, attach the new build to a version, fill in release notes, submit for review.

### Common review snags

- **5.2.5 (Apple trademarks)** — anything in the subtitle/keywords/screenshots that uses "Mac", "macOS", "App Store", "AirDrop", etc. as a product modifier. Use generic terms.
- **`network.server` entitlement scrutiny** — first MAS submission of v2.1 may prompt the reviewer to ask "why does a keep-awake utility need a server?" Reply in the Resolution Center: "Niacin runs a localhost-only MCP endpoint (bound to 127.0.0.1) so AI agents can request keep-awake assertions via the Model Context Protocol. All requests require a bearer token generated by the user in Settings. No network traffic leaves the device."

## Failure modes

| Symptom | Likely cause |
|---|---|
| Notarization rejected on `.app` | `xcrun notarytool log <SUBMISSION_ID> --keychain-profile niacin-notary` for the actual reason — usually a missing Hardened Runtime flag or a bad entitlement. |
| MAS archive upload fails on missing entitlements | Confirm `niacin/niacin.entitlements` is the file referenced by `CODE_SIGN_ENTITLEMENTS` in Build Settings; it must contain `com.apple.security.app-sandbox=true`. |
| Gatekeeper rejects the stapled artifact | The `.app` was modified after stapling. Re-archive from scratch. |
| MCP server can't bind on launch | Port 11473–11479 all taken by another local service. The server logs the chosen port via `log show --predicate 'subsystem=="com.oldsalt.niacin" AND category=="mcp"'`. |
